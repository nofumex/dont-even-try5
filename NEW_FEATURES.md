## Управление подпиской

### Описание
Администратор может включать/отключать требование подписки на канал перед использованием бота.

### Как использовать

1. Откройте админ-панель: `/admin`
2. Нажмите кнопку **🔒 Subscription ON** или **🔓 Subscription OFF**
   - Если подписка **вкл** (ON): пользователи должны подписаться на канал перед использованием
   - Если подписка **выкл** (OFF): пользователи могут использовать бот без подписки

3. Статус обновляется в реальном времени

### Технические детали
- Настройка сохраняется в файле `settings.json`
- По умолчанию подписка **включена** (true)
- Переменная окружения `.env`: `REQUIRE_SUBSCRIPTION=true|false` (опционально)

## Динамические категории товаров

### Описание
Администратор может создавать неограниченное количество категорий товаров (помимо стандартных "Аккаунты" и "Прокси") и управлять товарами внутри них.

### Пример структуры

```
📦 Основные категории:
├─ 🧾 Аккаунты
│  ├─ FB Marketplace
│  ├─ eBay
│  └─ ...
├─ 🧰 Прокси
│  ├─ SOCKS5 Germany
│  ├─ SOCKS5 USA
│  └─ ...
└─ 🍬 Конфеты (НОВОЕ)
   ├─ Шоколадные конфеты
   ├─ Конфеты без сахара
   └─ ...
```

### Как использовать

#### 1. Добавить новую категорию товаров

1. Откройте админ-панель: `/admin`
2. Нажмите **🗂 Manage catalog**
3. Нажмите **📦 Manage products**
4. Нажмите **➕ Add product category**
5. Введите название категории (например: `Конфеты`)
6. Введите иконку/эмодзи (например: `🍬`)

✅ Категория добавлена!

#### 2. Добавить товар в категорию

1. Откройте админ-панель: `/admin`
2. Нажмите **🗂 Manage catalog**
3. Нажмите **📦 Manage products**
4. Нажмите **✏️ Edit category items**
5. Выберите категорию
6. Нажмите **➕ Add item**
7. Введите детали товара в формате:
   ```
   Название | папка | цена [| флаг]
   ```
   Примеры:
   - `Шоколадные конфеты | candies_chocolate | 10`
   - `Конфеты без сахара | candies_sugarfree | 12 | 🍭`

✅ Товар добавлен!

#### 3. Изменить цену товара

1. Откройте админ-панель: `/admin`
2. Нажмите **🗂 Manage catalog**
3. Нажмите **📦 Manage products**
4. Нажмите **✏️ Edit category items**
5. Выберите категорию
6. Выберите товар
7. Нажмите **💲 Change price**
8. Введите новую цену

✅ Цена обновлена!

#### 4. Удалить товар из категории

1. Откройте админ-панель: `/admin`
2. Нажмите **🗂 Manage catalog**
3. Нажмите **📦 Manage products**
4. Нажмите **✏️ Edit category items**
5. Выберите категорию
6. Выберите товар
7. Нажмите **🗑 Remove item**

✅ Товар удален!

#### 5. Удалить категорию товаров

1. Откройте админ-панель: `/admin`
2. Нажмите **🗂 Manage catalog**
3. Нажмите **📦 Manage products**
4. Нажмите **🗑 Remove product category**
5. Выберите категорию для удаления

✅ Категория удалена!

### Загрузка файлов товаров

Файлы товаров загружаются через документы в Telegram в формате `.txt`.

**Важно:** имя файла должно содержать название папки категории!

Примеры:
- Для товара с папкой `candies_chocolate` отправьте файл: `candies_chocolate_account1.txt`
- Для товара с папкой `proxy_us` отправьте файл: `proxy_us_account1.txt`

Система автоматически определит категорию по названию папки в имени файла.

### Структура файла catalog.json

```json
{
  "categories": { ... },      // Старая система (Accounts)
  "proxies": { ... },         // Старая система (Proxies)
  "product_categories": {
    "Конфеты": {
      "icon": "🍬",
      "items": {
        "Шоколадные конфеты": {
          "folder": "candies_chocolate",
          "price": 10
        },
        "Конфеты без сахара": {
          "folder": "candies_sugarfree",
          "price": 12,
          "flag": "🍭"
        }
      }
    }
  }
}
```

### Для пользователей

Пользователи видят все категории в меню "🛍️ Products":
- Выбирают категорию
- Видят товары в категории
- Выбирают количество и покупают

Все работает так же, как со стандартными Аккаунтами и Прокси!

---

## Заметки

- Все категории (старые и новые) хранятся в `catalog.json`
- Папки товаров должны находиться в папке `data/` (например: `data/candies_chocolate/`)
- При загрузке товаров нужно создавать `.txt` файлы с данными в соответствующих папках
- Система поддерживает backward compatibility со старой структурой
