# 🔄 Changelog - Обновленные файлы и функции

## Дата: 2026-04-16

### 📝 Измененные файлы

#### 1. `config.py` - Добавлена настройка подписки
**Изменения:**
- Добавлена переменная `REQUIRE_SUBSCRIPTION` (загружается из `.env`)
- По умолчанию: `True` (подписка обязательна)

**Новый код:**
```python
REQUIRE_SUBSCRIPTION = os.getenv("REQUIRE_SUBSCRIPTION", "true").lower() in ["true", "1", "yes"]
```

---

#### 2. `database.py` - Системы настроек и динамических категорий
**Новые функции:**

**Настройки:**
- `load_settings()` - Загрузить настройки из `settings.json`
- `save_settings(settings)` - Сохранить настройки
- `get_require_subscription()` - Получить текущее значение требования подписки
- `toggle_require_subscription()` - Переключить требование подписки

**Динамические категории:**
- `get_product_categories()` - Получить все основные категории
- `add_product_category(name, icon)` - Добавить категорию
- `remove_product_category(name)` - Удалить категорию
- `add_item_to_category(cat_name, item_name, folder, price, flag)` - Добавить товар
- `remove_item_from_category(cat_name, item_name)` - Удалить товар
- `update_item_price(cat_name, item_name, price)` - Изменить цену
- `get_items_by_category(cat_name)` - Получить товары категории
- `get_item_info(cat_name, item_name)` - Получить информацию о товаре
- `get_item_info_by_folder_new(folder)` - Получить категорию и товар по папке

**Структура catalog.json:**
```json
{
  "product_categories": {
    "Категория": {
      "icon": "🎯",
      "items": {
        "Товар": {
          "folder": "item_folder",
          "price": 10,
          "flag": "🏴 (опционально)"
        }
      }
    }
  }
}
```

### ✨ Ключевые особенности

1. **Полная совместимость назад**
   - Старые категории (Accounts, Proxies) продолжают работать
   - Новые категории работают параллельно
   - Можно использовать обе системы одновременно

2. **Гибкая система категорий**
   - Неограниченное количество категорий
   - Каждая категория может иметь свою иконку
   - Каждая категория может содержать неограниченное количество товаров

3. **Удобное управление**
   - Все операции через админ-панель
   - Интуитивный интерфейс
   - Валидация ввода данных

4. **Безопасность**
   - Только администраторы могут управлять категориями
   - Проверки прав доступа на всех уровнях
   - Безопасная работа с файлами

### 📌 Примечания для разработчика

1. Все функции поддерживают исходную структуру catalog.json
2. Новая структура "product_categories" опциональна и совместима со старой
